Về Dự Án & Phương Pháp Đào Tạo

Intern sẽ làm việc trên use case dự đoán Churn có sẵn (CASA Churn) của NCB – không cần xây mới từ đầu mà tập trung vào việc sử dụng công cụ và hiểu quy trình DS end-to-end. Lộ trình chia làm 2 giai đoạn rõ ràng: Giai đoạn 1 (Tuần 1-2) tập trung vào truy vấn dữ liệu từ BigQuery, phân tích và tạo dashboard bằng Python. Giai đoạn 2 (Tuần 3-4) tập trung vào Feature Shortlisting, Univariate Analysis và huấn luyện model.

Mỗi tuần, Intern sẽ có một Sản phẩm bàn giao (Deliverable) cụ thể. Mentor hướng dẫn qua Daily Standup 15 phút mỗi sáng và kiểm tra tiến độ vào cuối chiều. Intern được cung cấp sẵn danh sách features và pipeline model mẫu để tập trung vào phân tích và hiểu bản chất, không phải tự dò dẫm.

Tuần 1: Kết nối dữ liệu & Khám phá (Data Access & Exploration)

Trọng tâm: Truy vấn dữ liệu thực tế CASA Churn từ BigQuery, làm quen cấu trúc bảng, thống kê tổng quan bằng Python
Ngày 1-2 Thiết lập môi trường & Kết nối BigQuery
  • Cài đặt môi trường Python (Jupyter/Colab), cài đặt thư viện 'google-cloud-bigquery' và xác thực tài khoản service account.
  • Viết câu truy vấn SQL đầu tiên trên BigQuery Console: 'SELECT * FROM dataset.table LIMIT 100' để xem cấu trúc dữ liệu CASA Churn.
  • Load kết quả truy vấn vào Python DataFrame bằng 'client.query(sql).to_dataframe()'. Kiểm tra 'df.shape', 'df.info()', 'df.dtypes'.
Ngày 3 Data Profiling & Kiểm tra chất lượng dữ liệu
  • Kiểm tra tỷ lệ missing values cho từng cột bằng 'df.isnull().mean()'. Đánh dấu các cột có tỷ lệ khuyết >30% cần xử lý đặc biệt.
  • Sử dụng 'df.describe()' để đọc phân phối số liệu. Đối chiếu với Data Dictionary do Mentor cung cấp để hiểu ý nghĩa nghiệp vụ từng cột.
  • Kiểm tra bản ghi trùng lặp 'df.duplicated().sum()', phân phối nhãn target (Churn vs Non-Churn) bằng 'value_counts(normalize=True)'.
Ngày 4-5 EDA cơ bản & Trực quan hóa đơn biến
  • Vẽ Histogram + Boxplot cho các biến số quan trọng (Số dư CASA hiện tại, Tỷ lệ thay đổi số dư, Độ tuổi, Số tháng không giao dịch) bằng Matplotlib/Seaborn.
  • Vẽ Countplot/Bar chart cho các biến phân loại (Giới tính, Nhận lương qua tài khoản, E-banking). Viết nhận xét nghiệp vụ bên dưới mỗi biểu đồ.
  • Đề xuất 3-5 câu hỏi kinh doanh từ dữ liệu (VD: "Khách hàng không nhận lương có tỷ lệ churn cao hơn không?", "Có mối liên hệ giữa số dư CASA và churn không?").

Tuần 2: Python Dashboard & Phân tích chuyên sâu

Trọng tâm: Tạo dashboard trực quan bằng Python (Plotly/Matplotlib), phân tích đa biến, đúc kết insight nghiệp vụ
Ngày 6-7 Xây dựng Python Dashboard cho Key Metrics
  • Tạo dashboard tổng hợp bằng 'matplotlib.pyplot.subplots()' hoặc 'plotly': Tỷ lệ Churn tổng thể, phân bổ theo Hạng thẻ, phân bổ theo Khu vực.
  • Vẽ so sánh phân phối các feature liên tục giữa 2 nhóm Churn=1 vs Churn=0 (overlapping Histograms hoặc Violin plots). Nhận xét feature nào có sự khác biệt rõ rệt.
  • Vẽ ma trận tương quan Heatmap bằng Seaborn. Xác định các cặp biến có tương quan cao (>0.7) cần lưu ý khi đưa vào mô hình.
Ngày 8-9 Phân tích chuyên sâu & Chân dung khách hàng Churn
  • Sử dụng 'groupby()' tính Churn Rate theo từng segment (Giới tính, Hạng thẻ, Khu vực & Số sản phẩm). Vẽ biểu đồ cột chồng (stacked bar).
  • Phân tích Bivariate: Scatterplot giữa Số lần giao dịch vs Giá trị giao dịch, tô màu theo nhãn Churn. Kết luận về hành vi chi tiêu.
  • Tóm tắt "3 dấu hiệu rõ nhất" của khách hàng có nguy cơ churn từ tất cả biểu đồ đã phân tích.
Ngày 10 Tổng hợp & Trình bày kết quả phân tích
  • Thiết kế slide 5 trang trình bày kết quả phân tích: Bối cảnh → Phân bổ Churn → Top 3 dấu hiệu → Chân dung KH rủi ro → Đề xuất hành động.
  • Thuyết trình nội bộ với Mentor. Nhận phản hồi và điều chỉnh trước khi chuyển sang giai đoạn Model Development.
OPTIONAL: Looker Studio (Data Studio) Dashboard
  • Export kết quả phân tích từ Python ra Google Sheets hoặc CSV, kết nối vào Looker Studio làm nguồn dữ liệu.
  • Tạo dashboard tương tác trên Looker Studio: Pie chart Churn Rate, Bar chart theo Hạng thẻ, Bộ lọc theo Khu vực/Giới tính. Chia sẻ link cho Mentor.

Tuần 3: Feature Selection & Tiền xử lý (Feature Shortlisting)

Trọng tâm: Nhận danh sách features có sẵn, thực hành Univariate Analysis, lọc biến quan trọng, xây pipeline tiền xử lý
Ngày 11-12 Tiếp nhận Feature List & Tìm hiểu nghiệp vụ từng biến
  • Nhận danh sách features có sẵn (pre-created feature list) từ Mentor. Đọc tài liệu mô tả ý nghĩa nghiệp vụ của từng biến.
  • Phân loại features thành các nhóm: Demographics (tuổi, giới tính), Transaction Behavior (số lần GD, giá trị GD), Product (tiết kiệm, thấu chi), Engagement (e-banking, tháng inactive).
  • Kiểm tra data quality của từng feature: tỷ lệ missing, tỷ lệ zero, phân phối outlier. Đánh dấu features cần xử lý trước khi đưa vào model.
Ngày 13-14 Univariate Analysis & Feature Shortlisting
  • Thực hành Univariate Analysis: Với mỗi feature số, tính KS Statistic hoặc dùng biểu đồ phân phối so sánh giữa nhóm Churn vs Non-Churn.
  • Với feature phân loại, tính Churn Rate theo từng category và sử dụng Chi-squared test hoặc WoE/IV (Weight of Evidence / Information Value) nếu có hướng dẫn.
  • Lập bảng tổng hợp Feature Shortlist: Xếp hạng các features theo mức độ phân biệt (discriminatory power). Loại bỏ features yếu hoặc bị multicollinearity.
Ngày 15 Xây dựng Pipeline tiền xử lý dữ liệu
  • Xử lý missing values ('SimpleImputer'), mã hóa biến phân loại ('OneHotEncoder'/'LabelEncoder'), chuẩn hóa biến số ('StandardScaler').
  • Tạo bảng so sánh "Before vs After": số lượng features ban đầu → sau shortlisting → sau tiền xử lý. Giải thích lý do loại/giữ từng feature.

Tuần 4: Huấn luyện Model & Trình bày kết quả

Trọng tâm: Train model với features đã shortlist, đánh giá hiệu năng, trích xuất insight, storytelling 10 phút
Ngày 16-17 Train Baseline Model & Đánh giá
  • Chia tập dữ liệu Train/Test (80/20) với 'train_test_split(stratify=y)'. Giải thích tại sao cần stratify khi dữ liệu mất cân bằng.
  • Nhận Pipeline model từ Mentor (Logistic Regression / Random Forest). Huấn luyện trên tập train, ghi lại AUC-ROC, Precision, Recall, F1-Score.
  • Vẽ ROC Curve và Confusion Matrix. Thảo luận với Mentor: Trong bài toán churn banking, nên ưu tiên Precision hay Recall? Tại sao?
Ngày 18-19 Feature Importance & Kiểm chứng kết quả
  • Trích xuất Feature Importance ('model.feature_importances_' hoặc 'model.coef_'). Vẽ biểu đồ horizontal bar chart Top 10 features quan trọng nhất.
  • Kiểm chứng: So sánh Feature Importance từ model với kết quả Univariate Analysis ở Tuần 3. Kết luận xem 2 góc nhìn có nhất quán không.
  • (Nâng cao) Thử thay đổi tập features (thêm/bớt) và so sánh AUC để chứng minh Feature Selection ảnh hưởng thế nào đến hiệu năng model.
Ngày 20 Storytelling & Báo cáo Demo cuối kỳ
  • Thiết kế bài thuyết trình "Storytelling" 10 phút: Bối cảnh nghiệp vụ → Phân tích dữ liệu (GĐ1) → Feature Selection → Kết quả Model → Đề xuất hành động cho phòng CSKH.
  • Demo trước Mentor (và stakeholder nếu có). Nhận phản hồi, tổng kết những gì đã học được trong 1 tháng.
OPTIONAL: Trình bày Model Results trên Looker Studio
  • Export kết quả dự đoán + Feature Importance ra Google Sheets, tạo dashboard kết quả model trên Looker Studio để stakeholder dễ theo dõi.
💡 Dashboard kết quả model giúp trình bày chuyên nghiệp hơn so với screenshot biểu đồ từ Jupyter Notebook.

Ngữ cảnh nghiệp vụ CASA Churn

Tại ngân hàng NCB, việc giữ chân khách hàng (Customer Retention) có chi phí tối ưu hơn nhiều so với việc tìm kiếm khách hàng mới. Số dư tiền gửi không kỳ hạn (CASA) là dòng vốn giá rẻ đặc biệt quan trọng giúp ngân hàng tối ưu hóa biên lợi nhuận lãi thuần (NIM) và cung cấp nguồn vốn dồi dào tài trợ cho các hoạt động tín dụng.
