Data Architecture Review

Priority Customer Data Pipeline

Tài liệu mô tả luồng xử lý dữ liệu từ KHUT (Phòng Khách hàng Ưu tiên), qua Data Platform, tới Decision Engine và giải pháp phân phối kết quả đầu ra.

SLA toàn trình: 3 ngày (Từ lúc KHUT gửi file đến lúc DP trả kết quả)

1. Luồng Dữ Liệu Hiện Tại

Đã hoàn thành

KHUT

Upload Data Portal

Phòng Khách Hàng Ưu Tiên

Thực hiện upload danh sách ngoại lệ qua cổng Data Portal.

SLA: 5 ngày đầu tháng

Data Platform

BigQuery (Minh & Chung)

Data Platform (DP)

Trước khi DE chạy:

  • Nhận dữ liệu upload từ KHUT
  • Full Join: Hành vi T.trước + Hạng T.trước + Ngoại lệ

Sau khi DE chạy:

  • Join kết quả từ DE để tạo bảng contract
  • Gửi lại kết quả cho KHUT (Chưa có luồng email)

Decision Engine

Scope Down Logic (Nguyệt)

Decision Engine (DE)

Đọc bảng dữ liệu input từ Data Platform.

  • Áp dụng điều kiện lọc (vài chục nghìn KH).
  • Chạy batch (vài trăm KH/lần).
  • Ghi kết quả vào BQ(DE) và DB(DE).

2. Giải Pháp Phân Phối Output

Mục tiêu: Trả về 2 file (Danh sách khách hàng ưu tiên & Chi tiết hợp đồng) cho phòng KHUT.

Decision Engine Tự Vận Hành

DE tự động sinh lệnh Create File trên Bucket → Call Bigquery Exec → Tạo Shared Link → Gửi Email cho KHUT.

Không khuyến nghị

Ưu điểm

Nhanh và liền mạch. Không cần thời gian chờ đợi vì hệ thống DE biết chính xác khi nào tiến trình xử lý xong để kích hoạt luồng xuất file tiếp theo.

Nhược điểm

  • Thiếu xử lý tạo danh sách chi tiết hợp đồng (nếu DE làm thì bị sai scope công việc, khó maintain).
  • Dữ liệu output không nằm trên Data Platform chuẩn để có thể tái sử dụng sau này.

Đồng Bộ Ngược (ETL) Về Data Platform

Luồng xử lý tuân thủ Policy bảo mật: DE chỉ có quyền ghi sang Project DE. Dữ liệu contract nằm ở Project DP. Không được cross-join.

Khuyến nghị
1

Job ETL Kéo Dữ Liệu

Xây dựng luồng ETL đồng bộ dữ liệu output (Danh sách phân hạng) từ Project DE về lại Project DP.

2

Join Contract Data (Single Source of Truth)

Tại Project DP, thực hiện join danh sách khách hàng vừa kéo về với bảng Contract Data để tạo ra thông tin chi tiết hợp đồng.

3

Xuất File & Phân Phối

Tạo 2 file CSV (Danh sách khách hàng & Chi tiết hợp đồng) và gửi qua email cho KHUT.

Kế Hoạch Thực Thi (Action Plan)

Cải tiến Nghiên cứu giải pháp tự động hóa luồng xuất file CSV và gửi Email từ DP.
